Trilium. Notas jerárquicas en Markdown

Lectura: 2 min. 250 palabras. markdown

Artículos  Podcasts  Tags

Quizás ha llegado el momento de tomar notas para seres humanos, voy a

hablar de Trilum.

Trilium es un método jerárquico para tomar notas, tareas,... donde estás

se gestionan con un método jerarquico como el orgmode y las podemos

exportar a posterior a HTML, Markdown o incluso imprimirlas en pdf. 

El método para introducir el texto es mediante la sintaxis markdown.

He probado diferentes dockers para la arquitectura amd64 y ARM así que finalmente me quedo con él dockers oficial para amd64 y otro que he encontrado de "hlince" para arm, que funciona muy bien en Raspberry.

Docker amd64

El docker oficial y más actualizado para amd64 es:

#+BEGIN_EXAMPLE

docker run -t -i --name=trilium -p 127.0.0.1:99:8080 -v ~/trilium-data:/root/trilium-data zadam/trilium

#+END_EXAMPLE

Servicio disponible por el puerto en este ejemplo 99.

Docker con Proxy Inverso con Caddy

1.- Pon almohadilla delante del volumen. Para iniciar el docker sin compartir volumen


      #volumes:
      #  - ./trilium-data:/home/node/trilium-data

2.- Una vez iniciado del docker, copia el volumen trilium-data en el host

docker cp trilium:/home/node/trilium-data ./trilium-data

3.- Descomenta quitando las almohadillas y vuelve a crear el docker con el directorio de trilium-data


version: '2.1'
services:
  trilium:
    image: zadam/trilium
    restart: always
    container_name: trilium
    ports:
      - "8080:8080"
    volumes:
      - ./trilium-data:/home/node/trilium-data
    networks:
      - DockerLAN


###########
## REDES ##
###########
networks:
  DockerLAN:  ## Red bridge propia ##
    name: DockerLAN
    driver: bridge
    ipam:
      driver: default
      config:
        - subnet: 172.20.0.0/24
