Docker: Wallabag

Lectura: 1 min. 192 palabras. docker

Artículos   Podcasts   Tags

Tiempo estimado de lectura: 2 minutos. 280 palabras. docker

Os hable de Wallabag en el Podcast, cuando estaba en la primera versión. Al pasar a la segunda versión, todo cambió. La interfaz mejoró muchísimo, más moderna, más opciones,… pero dejó de funcionar para Raspberry.

Ha pasado el tiempo y ahora que estoy liado con la creación de Dockers, así que decidí el intentarlo una vez más y esta vez con éxito.

Montando el Docker con docker compose

version: "3.7"
services:
  wallabag:
    image: wallabag/wallabag
    init: true
    container_name: wallabag
    environment:
      - SYMFONY__ENV__DOMAIN_NAME="https://wallabag.midominio.duckdns.org"
      - SYMFONY__ENV__SERVER_NAME="vps"
#    volumes:
#      - ./images:/var/www/wallabag/web/assets/images
#      - ./data:/var/www/wallabag/data

Logeate por primera vez:

  • usuario: wallabag
  • contraseña: wallabag

Hacer los volumenes persitentes

1.- Pon almohadilla delante del volumen. Para iniciar el docker sin compartir volumen

volumes:

  • ./images:/var/www/wallabag/web/assets/images
  • ./data:/var/www/wallabag/data

2.- Una vez iniciado del docker, copia los volumenes de wallabag en el host, dentro del directorio donde está el docker-compose.yml

docker cp wallabag:/var/www/wallabag/web/assets/images ./images docker cp wallabag:/var/www/wallabag/data ./data

Da permisos a ambos directorios sudo chmod 777 -R images/ data/

3.- Descomenta quitando las almohadillas y vuelve a crear el docker con el directorio de wallabag docker-compose up -d
