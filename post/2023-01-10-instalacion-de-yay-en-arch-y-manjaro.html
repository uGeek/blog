Instalación de yay en Arch y Manjaro

Yay es un asistente desarrollado en GO para instalar software de AUR (Arch User Repository).

El funcionamiento es muy sencillo, ya que si estás familiarizado con pacman, muchos comandos son muy parecidos.

Vamos a ver como instalarlo en Manjaro y ArchLinux, así como conocer muchos de los comandos disponibles.

Instalación

Puedes visitar el repositorio de Yay en el momento de la instalación por si ha cambiado algo, pero a continuación te muestro como instalarlo en Manjaro y Arch.

Manjaro


sudo pacman -S --needed git base-devel yay

Arch


sudo pacman -S --needed git base-devel && git clone https://aur.archlinux.org/yay.git && cd yay && makepkg -si

Primer uso

Genera una base de datos de paquetes de desarrollo para *-gitpaquetes que se instalaron sin yay. Este comando solo debe ejecutarse una vez.


yay -Y --gendb

Buscar actualizaciones del paquete de desarrollo


yay -Syu --devel

Hacer actualizaciones de paquetes de desarrollo de forma permanente activado ( yay=y =yay -Syu luego siempre verificará los paquetes de desarrollo)


yay -Y --devel --save

Comandos básicos

Actualizar base de datos paquetes disponibles


yay -Sy

Actualizar paquetes si hay actualizaciones


yay -Syu

Actualizar base de datos disponibles y actualizar apliciones


yay

Buscar un paquete


yay [Nombre del paquete]

Te mostrará todos los paquetes disponibles

Listar paquetes instalados con yay


yay -Qqm

Instalar paquetes con yay


yay -S [paquete]

yay typora

Borrar paquetes


yay -R [paquete]

Eliminar paquete y sus dependencias


yay -Rns [paquete]

Eliminar memoria caché

Cuando instalas nuevas actualizaciones, las versiones antiguas no son eliminadas. Si una vez probadas las nuevas actualizaciones, todo funciona correctamente, puedes eliminar las versiones antiguas almacenadas en la memoria caché con:


yay -Sc

Resumen de más opciones disponibles en el repositorio de yay

  • yay= > alias que =yay -Syu.
  • yay <Search Term> > Presentar menú de selección de instalación de paquetes.
  • yay -Y --combinedupgrade --save > Haga que la actualización combinada sea el modo predeterminado.
  • yay -Ps > Imprimir estadísticas del sistema.
  • yay -Yc > Limpie las dependencias innecesarias.
  • yay -G <AUR Package> > Descarga PKGBUILD desde ABS o AUR.
  • yay -Gp <AUR Package> > Imprima en la salida estándar PKGBUILD desde ABS o AUR.
  • yay -Y --gendb > Genere la base de datos del paquete de desarrollo utilizada para la actualización del desarrollo.
  • yay -Syu --devel > Realice la actualización del sistema, pero también verifique las actualizaciones del paquete de desarrollo.
  • yay -Syu --timeupdate > Realice la actualización del sistema y use el tiempo de modificación de PKGBUILD (no el número de versión) para determinar la actualización.
  • yay -Wv <AUR Package>= > Votar por paquete (Requiere configuración =AUR_USERNAME=y =AUR_PASSWORDVariables de entorno). (yay v11.3+)
  • yay -Wu <AUR Package>= > Dejar de votar por el paquete (requiere configuración =AUR_USERNAME=y =AUR_PASSWORDvariables de entorno) (yay v11.3+)
