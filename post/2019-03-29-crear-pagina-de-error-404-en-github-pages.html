Crear página de error 404 en GitHub Pages

Tiempo estimado de lectura: 2 minutos. 226 palabras. web

Después de cambiar la página de uGeek de Jekyll a Org-Bash-Blog, muchos de los artículos antiguos han cambiado la ruta de dirección web en el servidor y si llegas a ellas, quedas en una página de error de GitHub que no tiene relación con la web. Decidí crear una página personalizada de error 404 y que tras 3 segundos,tal como expliqué en este Post, redireccionará con el Index del blog. Pero…

¿Como se hace una página de error 404 en GitHub Pages?

Súper Fácil!!!!

Tan simple como crear una página .html en la raiz del repositorio y nombrarla como 404.html. Cada vez que alguien acceda a una página que esa no exista, GitHub Pages redirigirá el tráfico hacia esta página 404.html.
