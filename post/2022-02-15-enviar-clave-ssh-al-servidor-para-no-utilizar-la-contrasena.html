Enviar clave SSH al servidor, para no utilizar la contraseña

Tiempo estimado de lectura: 2 minutos. 286 palabras. ssh

Para conectarme a mis servidores utilizo SSH y es muy importante tanto por seguridad como por comodidad, no tener que introducir cada vez que me conecto la contraseña.

Esto lo hacemos intercambiando las claves entre cliente y servidor. Cada vez que un cliente quiere conectarse, el servidor verirfica que este cliente está en el listado de clientes permitidos y si es así, permite el acceso.

Vamos a ver como generar la clave pública del cliente para enviar la servidor y como enviarla.

Generar claves público privada

Primero generaremos las claves de nuestra máquina local:

ssh-keygen

Ahora ya tenemos una clave única que identificará nuestro cliente. Vamos a enviarla al servidor

Enviar clave pública al servidor

Enviar la llave publica al servidor es tan sencillo como:

ssh-copy-id -i ~/.ssh/id_rsa angel@192.168.1.100

Recuerda que en este ejemplo angel@192.168.1.100 es el usuario e IP de mi servidor.

El servidor nos pedirá la contraseña para conectarnos. Una vez introducida, el servidor guardará la llave pública en su listado de clientes permitidos y ya no nos volverá a pedir la contraseña nunca más.
