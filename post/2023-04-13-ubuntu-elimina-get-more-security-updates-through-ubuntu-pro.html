Ubuntu: Elimina Get more security updates through Ubuntu Pro

Tiempo estimado de lectura: 3 minutos. 394 palabras. ubuntu

Es posible que si has actualizado ubuntu, te aparezca este mensaje:

Get more security updates through Ubuntu Pro with 'esm-apps' enabled:

Este mensaje indica que para obtener más actualizaciones de seguridad, es necesario habilitar "esm-apps" en Ubuntu Pro. Para hacerlo, puedes seguir los siguientes pasos:

  1. Inicia sesión en tu cuenta de Ubuntu en https://ubuntu.com/pro.
  2. Obtén un token personal gratuito que incluye hasta 5 máquinas.
  3. Ejecuta el comando sudo pro attach tu-token-personal en la terminal de tu máquina Ubuntu.
  4. Actualiza tu sistema utilizando el comando sudo apt update && sudo apt upgrade.

Esto debería permitir que se descarguen e instalen las actualizaciones de seguridad necesarias, incluyendo las relacionadas con los paquetes mencionados en el mensaje de error.

Eliminar Mensaje

Otra opción sería eliminar el archivo /etc/apt/apt.conf.d/20apt-esm-hook.conf, que proporciona el gancho que llama a la generación del mensaje de marketing. Puedes hacerlo con los siguientes comandos:

Moveremos el archivo 20apt-esm-hook.conf al directorio que crearemos, por si queremos volver atrás:

mkdir -p ~/relocated_apt
sudo mv /etc/apt/apt.conf.d/20apt-esm-hook.conf ~/relocated_apt/.

Ten en cuenta que esto no garantiza que aparezca en las futuras actualizaciones si hay una nueva versión.

Ahora si quieres actualizar todo el sistema Ubuntu, puedes utilizar los siguientes comandos:

sudo apt list –upgradable
sudo apt-get dist-upgrade

También puedes actualizar un solo paquete utilizando el comando sudo apt install "paquete". Finalmente, si deseas instalar de forma automática las últimas actualizaciones de seguridad en Ubuntu Server, puedes utilizar los siguientes comandos:

sudo apt install unattended-upgrades
sudo dpkg-reconfigure unattended-upgrades

Ten en cuenta que las actualizaciones pueden reiniciar los servicios de Ubuntu Server, lo que puede afectar la operación crítica de la empresa, por lo que debes ser cuidadoso con este tipo de actualizaciones. Puedes ejecutar sudo unattended-upgrade o sudo unattended-upgrade -d para instalar las actualizaciones de forma automática.
