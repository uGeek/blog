Tunel SSH para conectar puertos remotos como si fuera local

Tiempo estimado de lectura: 3 minutos. 354 palabras. ssh

En este proceso lo más importante, es tener en el directorio .ssh de tu usuario root, los archivos de configuración para acceder al servidor remoto. Piensa que para desviar los puertos remotos del servidor a nuestro PC, es necesario tener permisos de super usuario y la conexión se hará mediante el usuario root.

 

Un truquillo si el PC es tuyo y no quieres complicarte configurando nuevamente el servidor SSH, sería copiar el directorio .ssh de tu usuario al del usuario root.

 

cp -r ~/.ssh /root/.ssh

 

 

Si utilizas Ubuntu, para cambiar al usuario root con:

sudo su

 

\ En el archivo de configuración config dentro del directorio .ssh, he añadido la conexión a mi vps con el nombre vps.  

 

Host vps
Hostname 111.222.33.444
IdentityFile ~/.ssh/id_rsa
IdentitiesOnly yes
User ubuntu
Port 22

 

Creando el tunel

Ahora vamos a enlazar un puerto remoto del vps con el puerto local.

 

sudo ssh vps -fNL 89:localhost:85

  \ El puerto local 89 lo conecto con mi vps al puerto 85.                                                                                               

  • f para ejecutar el comando ssh en segundo plano
  • N no ejecuta ningún comando en remoto
  • L especifica la conexión
